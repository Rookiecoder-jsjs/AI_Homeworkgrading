# AI 作业批改模块 — 产品需求文档 (PRD)

> **当前阶段：MVP v0.1.0**  
> 本版本聚焦核心批改闭环的功能验证，前端采用纯内联 CSS-in-JS 极简设计，无 UI 框架依赖。交互和视觉将在后续迭代中持续打磨。

## 一、产品概述

### 1.1 产品定位

AI 作业批改模块是一款面向 K12 教育场景的智能批改工具，利用大语言模型 qwen3.6-flash 实现秒级作业批改，提供 Socratic 引导式个性化反馈，打通"教师布置 → AI 批改 → 学生订正 → 教师复核"完整闭环。

### 1.2 核心价值

| 角色 | 痛点 | 解决方案 |
|------|------|----------|
| 教师 | 日均批改 150+ 份作业，主观题耗时 20 分钟/份 | AI 秒级批改 + 批量处理，人工仅需复核低置信度项 |
| 学生 | 提交后等 1-2 天才获反馈 | 即时批改，Socratic 引导式反馈促进主动思考 |
| 教学管理 | 难以追踪学生薄弱点 | 看板统计 + 待复核队列，精准定位问题 |

### 1.3 业务目标

- 教师单份作业批改耗时：**20 分钟 → 5 分钟以内**
- 学生对批改建议的有效互动率（查看解析、二次订正）：**15% → 40%+**
- AI 客观题批改准确率：**≥ 99%**
- 教师对 AI 辅助批改的接受度：客观题 **90%**，主观题目标 **60%+**（当前 35%）

## 二、用户分析

### 2.1 用户画像

**教师**：
- K12 各学科教师，一个班级约 50 人，通常负责 2-3 个班级
- 批改时间集中在 20:00-23:00（占 65%）
- 约 40% 的教师仍习惯纸质批改后拍照上传
- 对客观题 AI 批改接受度高（90%），主观题接受度低（35%）
- 约 20% 的教师在 AI 判错后会"全盘否定 AI 结果"

**学生**：
- 对"对/错"结果关注度 100%
- 对"为什么错"的解析点击率仅 15%（改进空间大）
- 需要即时反馈以完成错题订正

### 2.2 核心使用场景

| 场景 | 描述 | 频率 |
|------|------|------|
| 日常作业批改 | 教师布置作业 → 学生提交 → AI 批改 → 教师复核 | 每日 |
| 纸质作业批改 | 学生拍照上传 → OCR 识别 → AI 批改 | 每日 |
| 错题订正 | 学生查看反馈 → 思考 → 二次提交订正 → 重新批改 | 每周 |
| 批量批改 | 教师一键批改某作业所有未处理的提交 | 每日 |

## 三、功能范围

### 3.1 MVP 功能（本期已交付）

**教师端**：
- 创建作业（支持选择题、判断题、填空题、简答题、作文/证明题 5 种题型）
- **拍照出题**（上传试卷/课本照片，AI 自动识别题型、题目内容、参考答案并填入表单）
- 作业列表管理（筛选、删除）
- 查看某作业的所有学生提交（含低置信度/已确认统计标签）
- 批量 AI 批改（一键批改所有待处理提交）
- 智能复核页（高置信度项自动折叠 + ✅ 自动通过标记，教师仅需关注低置信度项）
- **一键确认高置信度**（批量通过所有置信度 > 0.9 的答案）
- 教师看板（5 个可点击交互卡片 + 入场动画，点击卡片跳转对应详情）
- 待复核队列（汇总所有低置信度未覆写项，按低置信度数量排序）

**学生端**：
- 查看待完成作业列表（含 5 种状态标签：去提交 / 等待批改 / AI已批改 / 已复核 / 已订正）
- 提交作业（逐题文本输入 + 每道题独立拍照上传，图片缩略图预览）
- **题目图片展示**（数学证明题、几何图等保留教师上传的原始图片，OCR文字在图片下方）
- 查看批改结果（动画分数环 + 逐题反馈 + 状态标签 + 等待批改提示）
- Socratic 引导式反馈（三级逐层展开：先思考 → AI提示 → 完整解析）
- 错题订正（仅展示错题，修改后自动重新批改）
- 所有页面均有返回按钮导航

**系统能力**：
- **出题 OCR**（教师拍照上传 → AI 识别题型/内容/答案/分值，自动填入表单）
- **答题 OCR**（学生逐题拍照 → 提取文本，支持手写体和 LaTeX 数学公式）
- AI 自动批改（规则引擎处理客观题 + AI 处理主观题，按题型分流）
- Socratic 反馈生成（追问/提示/类比，不直接给答案）
- 置信度三级机制（绿 >0.9 / 黄 0.7-0.9 / 红 <0.7，低置信度进待复核队列）
- 批量批改（教师端一键处理所有待批提交）
- **智能复核**（高置信度自动折叠+绿色边框，教师仅关注低置信度项）
- **题目原图保留**（数学证明题/几何图等，OCR文字不替代原图）

### 3.2 非本期功能（后续迭代）

- 完整用户认证与权限系统
- 多班级 / 多教师管理
- 学生薄弱点知识图谱
- 批改历史与趋势分析
- 批量导入/导出作业
- 语音批改反馈
- 教师自定义 rubric 模板
- 学生错题本聚合

## 五、核心流程

### 5.1 主流程

```
教师创建作业(文本/拍照OCR出题) → 发布 → 学生提交(文本/逐题拍照)
    → 教师批量/单次触发 AI 批改 → 学生查看反馈(三级逐层展开+题目原图)
    → 学生订正(仅错题) → 自动重批改 → 教师一键确认高置信度+逐题复核低置信度 → 完成
```

### 5.2 AI 批改子流程

```
获取学生答案 → 判断题型 →
  ├─ 客观题(选择/判断) → 规则匹配 → 高置信度(0.99)自动通过
  ├─ 填空题 → 精确匹配优先 → 失败则 AI 语义判断
  └─ 主观题(简答/作文) → AI 综合评价 → 生成 Socratic 反馈 → 标记置信度
```

### 5.3 置信度分流策略

| 置信度 | 处理方式 | 前端标识 |
|--------|----------|---------|
| > 0.9（绿） | 自动通过，教师可批量确认 | 绿色圆点 + "高置信度" |
| 0.7 - 0.9（黄） | 建议教师复核 | 黄色圆点 + "中置信度" |
| < 0.7（红） | 强制教师复核，进入待复核队列 | 红色圆点 + "低置信度" |

## 六、技术约束与设计原则

**MVP 设计原则：极简、可用、无冗余依赖**

| 项 | 选型 | 理由 |
|----|------|------|
| 前端框架 | React 18 + TypeScript + Vite | 类型安全，快速 HMR |
| 样式方案 | **纯内联 CSS-in-JS** | 零额外 CSS 框架依赖，快速原型迭代 |
| 动效方案 | Framer Motion | 轻量声明式动画，支持 stagger/spring |
| 路由 | react-router-dom v7 | 标准 React 路由 |
| 后端框架 | FastAPI (Python) | 异步支持，自动 OpenAPI 文档 |
| 数据库 | SQLite (WAL 模式) | 零配置，单文件部署，适合 MVP |
| AI 模型 | **qwen3.6-flash** | DashScope OpenAI 兼容 API，支持多模态 |
| 文件存储 | 本地文件系统（`uploads/`） | MVP 简化，无需对象存储 |
| 页面设计 | 极简卡片式布局，暖蓝 + 柔和绿双主色 | 信任感 + 成长感，无过度装饰 |
